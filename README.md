# Chaotic OS

I built Chaotic OS as a fun web experiment to see what it would look like if an operating system was completely broken on purpose. I've always loved that retro, cyberpunk hacker aesthetic, so I designed the whole thing using pure black backgrounds with bright neon green text. I didn't want to rely on any heavy external libraries or frameworks for this, so everything you see is built using just plain HTML, CSS, and some basic JavaScript I threw together in a caffeine-fueled coding session. 

The main page acts as the landing site for the OS. To make it feel unstable and glitchy, I wrote some custom CSS keyframe animations that constantly flicker and distort the main title. I also added some floating text in the background that moves across the screen to look like broken physics variables, like setting gravity to negative numbers. The core of this page is a simple CSS grid layout where I documented the fictional "anti-laws" of this universe—basically explaining to the user that normal physics don't apply here.

From there, I linked it to a second page called the Play Labs, which is where I actually put those broken physics to the test. It's split into four mini-experiments. The first one is an anti-gravity zone where I used a JavaScript mousemove event listener to make floating emojis actively repel away from your cursor. The second is a little chemical synthesizer where clicking buttons instantly manipulates the DOM to change the height and color of a liquid inside a beaker. 

The last two labs were just fun visual experiments. I managed to build a spinning, color-shifting 4D prism using nothing but CSS rotation transforms and hue filters. Finally, I added a "System Matrix" module, which is basically just a fake terminal log. I set up a JavaScript array full of dumb, random error messages (like running out of coffee), and every time you click the simulate button, it grabs a random string and injects it into the auto-scrolling log. Since it's all just basic frontend code, you don't even need a server to run it—you just double-click the HTML file and it works perfectly in the browser.
